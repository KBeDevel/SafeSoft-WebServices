AUTENTICACIÓN

Método: POST

Parámetros:

  • id
  • criteria
  • pass

Condiciones:

  • id: Debe ser enviado como base64. No puede ser nulo.
  • criteria: Puede tomar los valores [username, email, token]. No puede ser nulo.
  • pass: parámetro opcional sólo si el criterio es token. Es un hash tipo SHA256, codificado a base64

Ver más información acerca de hashing de cadenas de texto a SHA256.
Ver más información acerca de codificación de cadenas de texto a base64.
cURL: (Ejemplo)

    $ curl --data "id=dG9wb2NvbmFsZXJnaWE=&criteria=username&pass=ZTBlNjA5N2E2ZjhhZjA3ZGFmNWZjNzI0NDMzNmJhMzcxMzM3MTNhOGZjNzM0NWMzNmQ2NjdkZmE1MTNmYWJhYQ==" \
      https://pftsafesoft.000webhostapp.com/api/Auth/Authenticate.php
Se retornará una cadena de texto con formato JSON:

    // RESPUESTA CORRECTA:

    {
        "hasErrors" : false,
        "token" : "VMwx8AWlmKQ1PzG4xUbdIIGYeq1HHD5d" 
    }

    // RESPUESTA CON ERROR:

    {
        "hasErrors" : true,
        "error" : "Error message"
    }
